# W2 市场基准对照 · 模型 vs 去 vig 收盘价 · 2026-07-07

**目的**:给台账三个战略前提做有参照系的裁决——①五大 ~0.99 够不够;②在赛联赛 ~1.05 相对当地市场弱不弱;③"产品价值锚八月五大、在赛只趟坑"成不成立。
**性质**:只读、离线、零 provider calls、不改线上路径。工具:`scripts/run_w2_market_baseline_eval.py`;产物:`runtime/market_baseline_eval/`。

## 1. 方法

**模型侧**(全部本地缓存):完全复用 #193 协议(`_fit_offline_lambda_model` 5 特征 + train-only temperature,walk-forward 无泄漏)。
- 五大:Understat 缓存,跨季 2023→2024(train=2023 全季,val=2024 全季)。pooled 复现值 0.9895 与台账 0.9895 **精确一致**,证明 harness 复用无偏。
- 在赛联赛:API-Football Pro 缓存(fixtures + statistics xG),跨季 2024→2025,pooled 拟合(小联赛单独 train<200 不达 `MIN_LAMBDA_FIT_SAMPLE`,pooling 与 #193 pooled 五大做法一致),per-league val 切片。

**市场侧**:football-data.co.uk 收盘赔率(五大 mmz4281 分季文件,在赛联赛 new/ 累积文件;人工下载,零额度)。全部命中 **Pinnacle closing**(PSC*);去 vig 用 proportional(1/odds 归一),overround 1.030–1.046 已记录。

**join 质量**(硬门:CSV 赛果 ≠ 缓存赛果即丢弃):11 联赛 join 率 0.9982–1.0(MLS/阿甲均 1.0),result 冲突合计 1 条。别名表覆盖中超新旧队名(API-Football 旧赞助名 vs football-data 现名)等 40+ 映射。

**关键效度声明**:收盘价含临场信息(首发/伤停),模型是纯赛前视角,对模型偏严——实测该偏差仅 +0.002~+0.006(五大 closing vs 周中 Pinnacle:E0 +0.004、D1 +0.002、SP1 +0.006、I1 −0.001、F1 +0.003),比下表 gap 小一个量级,不影响结论。

**判定规则(先于看市场数字定下)**:gap = 模型 LL − 市场 LL(同批 fixtures)。gap ≤ 0.02 = 分析级达标;0.02–0.04 = 可用但需市场锚定;> 0.04 = 不可单独展示。"锚八月"成立当且仅当五大全部 ≤ 0.03 且在赛全部 > 0.04。

## 3. 发现

**F1 · 台账的在赛 ~1.05 跑错了模型。** S10/S11 复验走 `build_walk_forward_predictions`(手设常数先验 + 纯进球特征,缓存 xG 未入模型);#193 拟合协议此前从未在在赛联赛运行。同协议补跑后 pooled 1.0416(先验)→ **1.0278**(拟合)。"弱"的近半是"没拟合"。

**F2 · "五大 vs 在赛"二分法在联赛粒度不成立。** gap 排序:法甲 0.0125 < **挪超 0.0164** < 意甲 0.0262 < **MLS 0.0268** < 西甲 0.0277 < 英超 0.0294 < **德甲 0.0470** < 中超 0.0524 < 巴甲 0.0538 < 瑞典超 0.0551(阿甲见 F6,不入序)。被台账判"薄盘/二级源"的 MLS,模型-市场 gap 与意甲/西甲同档、优于英超。挪超(被判"弱")是全系统第二好;德甲(被判"够用")落入"不可单独展示"档,且拟合后比手设先验更差(1.0324 vs 1.0241——5 特征模型对德甲高方差进球环境系统性失配)。绝对 LL 的跨联赛比较只反映联赛熵(市场 LL 从挪超 0.913 到德甲 0.986 天然差 0.07),台账正是用这种比较得出了错误分层。

**F3 · 判定结果。** 按预注册规则:达标(≤0.02):法甲、挪超;需锚定(0.02–0.04):意甲、西甲、英超;不可单独展示(>0.04):德甲、中超、巴甲、瑞典超。**"锚八月"两个条件都被打破**(德甲 >0.04 破前件,挪超 ≤0.02 破后件)→ 战略锚应从"五大/八月"改为"联赛级 gap 门槛",与开赛月份无关。挪超现在就在赛、有 odds(S13 PASS)、gap 全系统第二——待命八月的机会成本是真实的。

**F4 · 当前模型对收盘价零增量信息。** blend(p ∝ p_model^(1−w)·p_market^w,w 在 train 拟合)在 **10/11 联赛 val 上没有跑赢纯市场**:8 个 w*=1.0(含 MLS);德甲 w*=0.9、挪超 w*=0.75 属 train 过拟合,val 上仍略差于纯市场;唯一"例外"阿甲(w*=0.95,blend 略优于市场但劣于纯模型)属 F6 子集假象,不构成反证。直接推论:(a) 展示层若要市场级概率,应取 w≈1(详见提案文档);(b) **formal/+EV 腿在当前模型上不应开启**——模型无增量信息时,+EV 检出≈模型误差(赢家诅咒),3.5% 阈值挡不住系统性 0.03–0.055 的校准差。

**F5 · 冷启动比预期温和,但市场仍全程领先。** 赛季前 6 轮(跨季状态携带下):PL cold 模型 0.9270 vs 市场 0.9277(打平)、挪超 0.9753 vs 0.9779(略优);但 warm 段市场全联赛稳定领先。模型真正的冷启动悬崖不在"赛季初轮次",而在"零历史"场景(新启用联赛、世界杯)——那里只有市场锚定能立即给出可用概率。

**F6 · 阿甲"模型跑赢市场"是子集假象;MLS"薄盘"是供给问题。** 阿甲名义 gap −0.0115,尽调不过:①模型可评估子集(xG 齐 + min_history 热身后,n=242/465)的市场 LL 为 1.1063,而**全季市场 LL 仅 1.0682**——子集爆冷偏重,市场在该子集连 uniform(1.0986)都不如,被比较对象本身异常;②手设先验(1.0924)同样"赢"市场且赢拟合模型,无信号一致性;③n=242 下 ±0.01 级差异在噪声内。裁定:观察档,不授予达标,待前向数据(路线图 R1)重裁。方法论教训:同一 fixture 集内比较的设计正是这次识别出子集偏置、避免错误庆祝的原因。MLS 结论干净:football-data 有完整 Pinnacle 收盘(全季 n=525,市场 LL 1.0267),"薄盘"是 API-Football 的供给缺口,不是市场不存在;模型 gap +0.0268 居全体中游、优于英超。二级 odds 源优先级据此再降。

## 4. 限制

单一 val 季(2024/25、2025);proportional devig(power/Shin 会略降市场 LL,即 gap 只会更大,不利结论方向不变);closing 偏严已量化(≤0.006);中超/瑞典超/阿甲 n≈200–240 置信区间宽(gap ±0.03 级别的差异在这几个联赛间不应过度解读,但"与市场差 0.05"级别的主结论显著);模型可评估子集 ≠ 全季(阿甲案例表明子集可带结果偏置,gap 的号志意义以 |gap|>0.03 为宜,临界值交前向数据)。

## 5. 对台账结论的裁决汇总

| 台账结论 | 裁决 | 依据 |
|---|---|---|
| "五大 ~0.99 已够用、模型收益递减" | **不成立** | gap 0.013–0.047,系统性落后市场;德甲 >0.04;5 特征模型类的天花板,不是建模的天花板 |
| "在赛联赛 ~1.05 弱、五大结论不可迁移" | **测量错误 + 粒度错误** | 跑的是手设先验;拟合后 1.028;挪超 gap 全系统第二,德甲落入最差档 |
| "产品价值锚八月五大" | **不成立** | 正确锚 = 联赛级 gap 门槛;挪超/法甲现在达标,德甲八月也不达标 |
| "分析级不追求跑赢市场" | 成立,但要认账 | 那就该用市场级概率展示(w≈1 锚定),当前模型概率既不赢市场也不该假装等价 |
| "+EV 腿(3.5% 阈值)可在模型达标后开" | **当前禁开** | 模型对收盘零增量(F4);重开条件应为前向 CLV 证明,非离线 LL |
