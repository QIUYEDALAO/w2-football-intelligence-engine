# W2 市场锚定提案 + 前向资产积累 MVP · 2026-07-07

**性质**:提案,未实现。不改线上路径、不 enable、不 production。所有"实测"数字来自 `W2_MARKET_BASELINE_EVAL_2026_07.md`(同批 fixtures、Pinnacle 收盘、去 vig)。

## 1. 这是产品身份决定,不是调参

评审原设想是"市场先验 + 模型残差微调"。**实测修正了这个设想**:blend 权重在 train 上拟合后,9/9 联赛 val 上没有任何一个跑赢纯市场(6 个 w*=1.0;两个 w*<1 的在 val 上仍略差)。当前 5 特征模型对收盘价的增量信息 ≈ 0。

因此市场锚定的诚实形态不是"模型微调市场",而是**分层**:

- **概率层 = 市场**(有可用盘口时):展示概率取去 vig 收盘/最新盘口,概率即市场级(挪超 0.913 … 德甲 0.986),冷启动、新联赛、跨联赛一致性问题全部消失。DecisionCard 增 `probability_source: MARKET_DEVIG | MODEL_FALLBACK | BLEND` 与 `model_market_divergence` 字段——不动 DecisionCard/DayView 唯一数据源契约,只是加 provenance。
- **模型层 = 解释与分歧标注**:模型不再假装给同级概率,改供两件事——(a) 分歧雷达:|p_model − p_market| 超阈值的场次标"值得人看"(注意:在模型通过前向验证之前,分歧的语义是"模型与市场不一致",不是"市场错了";雷达可信度靠 §3 的 CLV 前向证据升级);(b) reason codes:哪些因子驱动模型观点(现有 feature 框架直接复用)。
- **回退层**:无盘口(世界杯赛前早窗、薄市场)→ 纯模型 + 显式降级标注(`MODEL_FALLBACK`,置信降档)。这取代"假装模型概率与市场同级"的现状。
- **EV 腿:保持关**。模型对市场零增量时,+EV 检出≈模型误差。重开条件(预注册):某联赛前向 ≥200 张卡的 CLV 中位数 > 0 且 blend 在滚动 val 上 w*<1 并稳定优于纯市场 ≥0.003。离线 LL 不再作为 EV 开关依据。

**与 `FORBIDDEN_MARKET_FIELDS` 教条的关系**:该门禁保护的是"独立模型不得偷看市场"——EV 腿必须保留此性质,否则自我对赌。概率展示层是模型下游的产品层,不受该门禁约束;实现上双轨输出(model_probs 与 display_probs 分字段),门禁继续对 model_probs 生效。

## 3. 前向 outcome ledger + odds 时间线 MVP(建议今天开始)

两类只增资产,都只能用日历时间换;每闲置一周 = 永久少一周。

**范围**:S13 odds-PASS 的在赛 4 联赛(巴甲/中超/瑞典超/挪超)+ 世界杯。staging only,不 enable production,不改决策路径——只记录。

**采集**(受控刷新护栏内):每日两个批次(T-24h 与 T-1h 各一次;世界杯加 T-15m):fixtures + odds 快照。额度估算:≤40 场/日 × ≤3 快照 × 1 call ≈ **≤120 calls/日**(Pro 7500 的 1.6%)。

**存储**:`runtime/forward_ledger/YYYY-MM-DD/{decision_cards.jsonl, odds_snapshots.jsonl}`,append-only,卡片哈希复用现有 audit;赛后回填 outcome。

**评分**:两条,都不需要大样本等待——
1. **CLV**:T-24 快照概率 vs 收盘概率。拣选质量在数百样本即可判(方向:被推荐侧的收盘移动)。这是 EV 腿重开的唯一合格证据源。
2. **outcome 校准**:滚动 LL/ECE,对 `probability_source` 分轨(市场轨 vs 模型回退轨),自动积累"模型 vs 市场"前向对照——把本次离线 eval 变成常设监控。

**顺带修复**:`bookmaker_intent`(线动启发式,当前驱动 ANALYSIS_PICK 方向)从未回测且无数据可回测——本 MVP 的 odds 时间线正是它的回测数据集。两个月后即可裁决该启发式的去留。

**验收(预注册)**:两周后 ≥100 张卡带 T-24/closing 双快照与 outcome;CLV 分布报告可生成。

## 4. 产品叙事变化(配合 S17,但先于视觉稿)
